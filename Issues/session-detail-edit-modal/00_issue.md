# 세션 상세 — 편집·재생성 모달

## 배경

세션 상세(`SessionDetail`)는 [session-detail-home-note-ui](../session-detail-home-note-ui/00_issue.md)에서 홈과 동일한 `RecorderNoteWorkspace` 패턴으로 정렬되었지만, 인터랙션은 여전히 *항상 편집 가능*한 인라인 형태다. 그 결과:

- 속성 인풋(`참석자/주제/키워드/요약 형식`)이 늘 활성이라 포커스만 들어가도 편집된다.
- 스크립트 탭이 `textarea` 그대로 노출되어 *문서를 본다*기보다 *편집 화면이 떠 있다*는 인상을 준다.
- 요약 생성 컨트롤(용어집·요약 모델·생성 버튼)이 스크립트 탭 하단에 있어 결과(AI 요약 탭)와 컨트롤(스크립트 탭) 위치가 어긋난다. 이 어긋남이 AI 요약 탭의 *"스크립트 탭 하단에서 요약을 생성하세요"* 안내 문구를 강제한다.
- 페이지 최상단에 *오디오 다운로드 버튼* 하나만 단독 액션바로 떠 있어 시각 비례가 맞지 않는다.
- 스크립트 STT 모델 표시(`SessionScriptMetaDisplay`)가 "요약 생성" 섹션 안에 들어가 있어 라벨과 위치가 어긋난다.

세션 상세의 실사용 빈도는 *대부분 읽기*이고, 편집·재생성은 부수 작업이다. 클로바 노트·Granola·Notion AI도 *읽기 표면을 정돈*하고 편집은 명시적 진입으로 분리한다.

본 이슈는 **세션 상세 페이지를 읽기 전용 표면으로 정돈**하고, **편집·재생성은 모달 다이얼로그**로 분리한다. 모달 패턴은 프로젝트의 정형인 `src/components/settings-panel.tsx`(`role="dialog"`, `aria-modal="true"`, 배경 클릭 닫기)를 그대로 따른다.

## 목표

1. **세션 상세 페이지를 읽기 모드 표면으로 정렬**한다.
   - 노트 제목: 현재처럼 `h2` 읽기 전용 유지.
   - 속성 행(`참석자/주제/키워드/요약 형식`): *값만* 렌더한다. 인풋이 활성화되지 않는다.
   - 스크립트 탭: 플레인 텍스트로 렌더한다(편집 불가). 스크립트가 비어 있으면 기존 안내 유지.
   - AI 요약 탭: 마크다운을 현재처럼 렌더한다.
   - 페이지에서 *"요약 생성" 섹션을 완전히 제거*한다. 모든 편집·재생성 컨트롤은 모달로 옮긴다.

2. **헤더 우측에 "편집" 트리거**(`Pencil` 아이콘 + 라벨)를 둔다. 클릭 시 편집 모달이 열린다.

3. **편집 모달**(이하 *세션 편집 다이얼로그*)을 신설한다.
   - 패턴: `SettingsPanel`과 동일 — `role="dialog"` + `aria-modal="true"` + 배경 클릭/ESC로 닫기 + 모바일은 bottom sheet, 데스크탑은 가운데 패널.
   - 모달 헤더: 제목 (예: *"노트 편집"*) + 닫기 버튼.
   - 모달 본문 구성:
     - 회의 정보: 참석자 / 주제 / 키워드 / 요약 형식 (인풋 활성).
     - 스크립트 편집: `textarea` (단일 편집 영역, 폭·높이는 모달 크기에 맞춰 조정).
     - 요약 도구: `SessionGlossaryEditor` + `SessionMinutesModelSelect` + 요약 생성/재생성 버튼 + 에러.
   - 모달 푸터: [닫기] / [저장] 액션. 저장 흐름은 아래 **확정된 결정** 절을 따른다.

4. **모달의 dirty·저장·닫기 정책**(아래 **확정된 결정** 절):
   - 변경 추적: 모달 내부 폼은 dirty 추적. 닫기 시 dirty면 확인 다이얼로그.
   - 저장: [저장]을 누르면 IDB(`updateSession`) 반영 후 모달 자동 닫힘. 페이지 상태는 `onSessionRefresh()`로 갱신.
   - 요약 생성/재생성: 클릭 시 *즉시 저장+생성*. 진행 중에는 **모달을 즉시 닫고** 페이지(헤더·탭 영역)에 진행 표시를 노출한다(백그라운드 진행).

5. **오디오 다운로드 위치**: 페이지 헤더 액션바(노트 제목 우측 영역)에 둔다. 모달과 무관.
   - 후보 동선: 헤더 우측 `[오디오 다운로드] [편집]` 순. 오디오가 없으면 다운로드만 미노출.

6. **스크립트 STT 모델 표시**(`SessionScriptMetaDisplay`): 페이지의 *스크립트 탭 상단 작은 부가 라벨*로 둔다(읽기 모드 페이지). 모달 안에는 두지 않는다.

7. **AI 요약 탭 빈 상태 안내** 문구를 새 동선에 맞게 갱신한다 — 예: *"아직 요약이 없습니다. 우측 상단 '편집'에서 요약을 생성하세요."*

8. **`useBeforeUnload(mmLoading)` 보호는 유지**한다. 모달이 닫혀 있어도 `mmLoading=true`이면 페이지 이탈 경고가 동작한다.

9. **`RecorderNoteWorkspace`는 홈 녹음 화면용 그대로 둔다**. 세션 상세는 *속성 행*을 별도 컴포넌트로 추출해 **읽기 전용 표시**와 **모달 내 편집 폼**에서 각각 재사용한다(아래 **확정된 결정** 절).

10. **홈 녹음 화면(`Recorder`)의 인터랙션 모델은 변경하지 않는다**.

## 비목표

- 요약/스크립트 **히스토리** 기능은 본 이슈에서 다루지 않는다. 데이터 모델(`Session` 스키마 확장), IndexedDB 마이그레이션, 히스토리 UI(목록·복원·삭제·용량 정책)는 별도 후속 이슈(`session-detail-summary-history` 가칭)로 트래킹한다.
- IndexedDB·파이프라인·회의록 API 계약은 바꾸지 않는다.
- 새 녹음 시작·`RecordingCard` 도킹을 세션 상세에 도입하지 않는다([session-detail-home-note-ui](../session-detail-home-note-ui/00_issue.md) 비목표 유지).
- 인앱 오디오 미리듣기 플레이어는 추가하지 않는다.
- 노트 제목 편집은 도입하지 않는다(현재처럼 읽기 전용 `h2` 유지).
- 모달 *내부에서* AI 요약 결과를 함께 보여주는 기능(편집-결과 분할 보기)은 본 이슈에서 다루지 않는다. 결과 확인은 페이지에서 한다.

## 트레이드오프 (수용)

- 모달이 페이지를 덮으므로, *편집 중에 현재 AI 요약 결과를 나란히 보기*는 어려워진다. 결과를 보고 싶으면 모달을 닫는다. 이 트레이드오프는 *읽기 표면의 정돈*과 *명시적 편집 진입*의 가치로 수용한다.
- 페이지에서 곧바로 한 단어 고치는 동선은 한 단계 길어진다(클릭 → 모달 → 편집 → 저장 → 닫힘). 본 화면이 *대부분 읽기* 빈도라는 전제하의 의도된 비용.

## 테스트·완료 조건

1. **페이지는 읽기 전용**:
   - 속성 행이 *값만* 렌더되고 인풋이 존재하지 않거나 `readOnly`/`disabled`.
   - 페이지에 `textarea`가 존재하지 않거나 `readOnly`.
   - 페이지에 "요약 생성" 섹션·도구 패널이 *없다*.
   - 스크립트 탭 상단에 *스크립트 모델* 라벨이 표시(메타가 있을 때).
   - 헤더 우측에 `[오디오 다운로드][편집]` 액션. 오디오 없으면 다운로드 미노출.
2. **모달 열기·닫기**:
   - "편집" 클릭 → `role="dialog"` + `aria-modal="true"`인 다이얼로그가 열린다.
   - 배경 클릭/ESC/[닫기] 버튼으로 닫힌다(*dirty 시 확인* 정책 작동).
3. **모달 내부 편집**:
   - 회의 정보 인풋 활성, 입력 가능.
   - 스크립트 `textarea` 편집 가능.
   - 용어집·요약 모델 변경 가능.
4. **저장 동작**:
   - [저장] 클릭 → `updateSession`이 호출되고 IDB·페이지 상태가 갱신되며 모달이 닫힌다.
   - 저장 실패 시 사용자 메시지 노출, 모달은 닫히지 않는다.
5. **요약 생성/재생성**:
   - 클릭 시 *즉시 저장+생성* 흐름이 시작된다.
   - 진행 중: **모달 즉시 닫힘**, 페이지에 진행 표시(백그라운드).
   - 생성 후 페이지 AI 요약 탭에 결과 반영.
6. **`useBeforeUnload(mmLoading)`** 동작이 모달 열림/닫힘과 무관하게 기존과 동등.
7. 오디오가 있을 때 ZIP 다운로드 동작 유지(실패 시 기존처럼 조용히 복구). 오디오 없을 때 다운로드 버튼 미노출.
8. **품질 게이트**: `npm test`, `npx tsc --noEmit`, `npx eslint .`, `npx prettier --check .`, `npm run build` 통과.
9. 홈 녹음 화면(`Recorder`)·기존 도킹·녹음 흐름·post-recording pipeline에 회귀 없음.

## 확정된 결정 (구현 전 확정됨)

- **모달의 크기/표면 형태**: **(b)** 대형 다이얼로그 — 데스크탑 `max-w-2xl`~`max-w-3xl` 수준으로 스크립트 편집 폭 확보. 모바일은 `SettingsPanel`과 동일한 bottom sheet 패턴 유지.

- **저장 흐름**: **(c)** 일반 필드 변경은 **[저장]**으로만 IDB 반영 후 모달 닫힘. **[요약 생성/재생성]** 클릭 시에는 *암묵적 저장* — 현재 폼 값을 먼저 `updateSession`에 반영한 뒤 요약 파이프라인을 시작한다.

- **dirty 상태에서 모달 닫기**: **(a)** 확인 다이얼로그 — *"저장하지 않은 변경이 있습니다. 닫으시겠어요?"* 등. 폐기·저장·취소 등 명시적 선택.

- **요약 생성/재생성 진행 중 모달 처리**: **(b)** 모달 **즉시 닫기**. 페이지(헤더 또는 탭 상단 영역)에 **진행 표시** 노출. 백그라운드로 생성 완료까지 진행. `useBeforeUnload(mmLoading)`은 기존과 동등하게 유지.

- **워크스페이스 readOnly 분기 위치**: **(c)** `RecorderNoteWorkspace`에서 **속성 행(Notion 스타일 행)**을 별도 컴포넌트로 추출. 세션 상세 페이지는 **읽기 전용 속성 블록**을 사용하고, 편집 모달은 **편집 가능한 동일 레이아웃** 컴포넌트를 사용한다. 홈 `Recorder`는 기존처럼 워크스페이스 인라인 편집 유지.

- **트리거 라벨·아이콘**: **`편집`** 텍스트 + **`Pencil`** 아이콘 (`lucide-react`).

- **모달 트리거 키보드 단축키**: **본 이슈에서 제외** (후속 검토).

## 후속 이슈 (분리)

- `session-detail-summary-history` (가칭): 요약 재생성 시 기존 결과를 보존하고 히스토리로 추가하는 기능. `Session` 스키마 확장, IDB 마이그레이션, 히스토리 UI·정책 포함. 모달 안에서 *덮어쓰기/새 버전 추가* 선택 토글로 자연스럽게 통합 가능.
