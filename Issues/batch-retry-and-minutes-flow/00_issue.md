# 배치 녹음 STT 재시도 및 회의록 파이프라인 연동

## 배경

"녹음 후 스크립트"(배치) 모드 사용 중 네트워크가 끊기면 중간 세그먼트 전사가 실패하고, 이후 회전 시점에 이전 실패를 다시 보내지 않아 스크립트가 잘린다. 녹음 종료 시 `stopAndTranscribe`가 실패하면 세션 저장·`enqueuePipeline`이 스킵되어 `/api/meeting-minutes`가 호출되지 않는다. 수동 "다시 시도"는 `retryTranscription`이 문자열만 반환해 Recorder가 파이프라인을 이어갈 수 없다.

## 요구사항

1. **자동 재시도 트리거**: 5분 세그먼트 회전 시점 + 브라우저 `online` 이벤트 복구 시, 이전에 실패한 세그먼트를 **시간순 엄격 순차**로 다시 `POST /api/stt/transcribe`에 보낸 뒤 현재 세그먼트를 전송한다.
2. **재시도 상한**: 없음 (녹음 종료 전까지 계속 시도).
3. **녹음 중 UI**: 실패가 있으면 작은 배지로 `N개 재시도 대기 중`만 표시 (조용한 피드백).
4. **녹음 종료 후**: 실패가 남으면 새로고침 아이콘 + 실패 개수 + 진행 시 `재시도 중… (k/N)` 형태의 "다시 시도" 버튼을 노출한다.
5. **회의록**: 모든 세그먼트 전사가 성공한 뒤 녹음이 끝나면, 자동/수동 재시도 경로와 무관하게 `meeting-minutes` API가 호출되도록 `enqueuePipeline`까지 이어진다.
6. **세션 저장**: 재시도가 모두 성공한 시점에만 `saveSession` (부분 저장 없음).
