# Feature 16: 녹음 완료 후 AI 회의록 작성

## 목표

녹음이 완료되면(기존 녹음 후 파이프라인과 동일한 타이밍) 전사 텍스트를 바탕으로 **회의록**을 생성한다. 현재 앱의 "요약" 슬롯(`summary`)을 회의록 용도로 채우거나, 회의록 전용 필드로 명확히 할 수 있다(구현 시 데이터 모델과 UI 용어를 일치시킬 것).

## 기능 요구사항

1. **모델**: 기본값은 **GPT-5.4 nano**(`gpt-5.4-nano` 등 API 식별자 — 실제 배포 가능한 모델 id로 매핑)를 사용한다.
2. **설정**: 녹음/전사에서 `batchModel`을 고르듯, **회의록 생성 모델**을 설정에서 바꿀 수 있어야 한다(동일한 패턴: 타입 + 기본값 + localStorage 등 기존 설정 저장 방식 재사용).
3. **긴 회의**: 전사가 아무리 길어도 **중간 내용을 누락하지 않도록** 회의록이 완성되어야 한다. 단일 프롬프트에 전체를 넣는 방식만으로는 부족할 수 있으므로, **청크/계층 요약(map-reduce), 누적 구조화, 또는 동등한 신뢰성 있는 전략**을 코드와 테스트로 뒷받침할 것.
4. **서버**: API 키는 클라이언트에 노출하지 않고, 기존 `/api/summarize`와 같이 **서버 라우트**에서 OpenAI(또는 프로젝트가 쓰는 공식 SDK)로 호출한다.

## 비기능

- 기존 파이프라인(전사 → 요약/회의록 → 세션 저장) 실패 시 동작·에러 처리 패턴 유지.
- `npm test`, 타입·린트·빌드 통과.

## 완료 조건

- 설정에서 회의록 모델 변경 가능.
- 녹음 종료 후 전사가 준비되면 선택한 모델로 회의록 생성.
- 매우 긴 입력에도 누락 최소화 전략이 구현·검증됨(단위 테스트 또는 통합 테스트).
