# 배치 전사 시스템 개선 (Phase 2)

## 1. 마지막 청크 누락 및 전사 결과 합산 버그 수정

- **문제**: `stopAndTranscribe` 시점에 진행 중인 백그라운드 전사 Promise들을 기다리지 않아 결과가 누락됨.
- **해결**: 진행 중인 전사 Promise들을 `Set`으로 관리하고, 종료 시 `Promise.allSettled`로 대기.

## 2. 1시간 녹음 제한 완화

- **내용**: `BATCH_HARD_LIMIT_MS`를 60분에서 4시간(240분)으로 상향 조정. 세그먼트 방식이므로 기술적 제약 없음.

## 3. 전사 진행률 UI 추가

- **내용**: 총 세그먼트 수 대비 완료된 전사 수(`completedCount / totalCount`)를 추적하여 UI에 표시.

## 4. 단일 오디오 파일 저장 (MediaRecorder 유지)

- **내용**: `MediaRecorder`를 중지하지 않고 `requestData()`를 사용하여 5분마다 청크를 추출. 최종 저장 시 모든 청크를 하나의 Blob으로 병합.

## 5. UX 개선 (5-1, 5-2, 5-4)

- **5-1. 실시간 부분 전사 표시**: 녹음 중에도 완료된 세그먼트의 전사 결과를 즉시 화면에 노출.
- **5-2. 세그먼트별 재시도**: 실패한 세그먼트만 식별하여 개별 재시도 가능하도록 로직 개선.
- **5-4. 인라인 오디오 플레이어**: 세션 상세 페이지에서 다운로드 없이 즉시 재생 가능한 플레이어 추가.
