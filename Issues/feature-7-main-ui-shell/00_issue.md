# Feature 7: 메인 UI 셸 — 브랜딩, History, 전사·요약 탭

## 1. 개요

메인 녹음 화면의 정보 구조와 반응형 레이아웃을 정리한다. 헤더 영역(Whirr 등)과 별도로, 메인 본문에서는 Record 아래에 **실시간 전사 텍스트** 탭과 **요약** 탭으로만 이루어진 탭 UI를 둔다. 데스크톱에서는 좌측 History 사이드바, 모바일에서는 오버레이 drawer로 동일 목록에 접근하게 한다.

## 2. 상세 기획 (Detailed Plan)

### 2.1 헤더 · 브랜딩

- 상단 **중앙**에 앱 이름 **Whirr**를 **텍스트만** 표시한다.
- 클릭·링크·홈 이동은 하지 않는다.
- **뷰포트 768px 미만**: 좌측 상단에 History drawer **트리거(예: 햄버거)** 를 두고, Whirr는 헤더 중앙에 배치해 트리거와 시각적으로 겹치지 않게 한다.

### 2.2 녹음 영역 순서 · 탭 위치

- 주 영역에서 **Record(녹음) 컨트롤**이 **위**에 온다.
- 그 **아래**에 **탭 UI**를 둔다. 탭은 **「실시간 전사 텍스트」** 와 **「요약」** 두 개뿐이며, **실시간 전사 내용은 전사 탭 패널 안**에서만 본다(헤더와 분리된 본문 플로우).

### 2.3 History · 사이드바 · drawer

- **768px 이상**: 화면 **왼쪽 사이드바**에 History를 표시한다. 데이터·범위는 **현재 세션 목록(`session-list` 등)과 동일**하게 유지한다.
- **768px 미만**: 동일 History를 **drawer**로 표시한다.
  - **표시 방식**: **오버레이**(본문 위에 덮음).
  - **라우트 이동 시** drawer는 **닫힌다**.
- 구현 시 기존 `src/components/home-content.tsx`, `session-list.tsx`, `src/app/page.tsx` 등 레이아웃을 조정한다.

### 2.4 실시간 전사 텍스트 탭 · 요약 탭 (항상 2탭, 본문 전용)

- **위치**: Whirr·History 트리거가 있는 **헤더 아래**, **Record 아래**의 메인 콘텐츠 영역에만 탭을 둔다. 앱 상단 전체 너비의 탭 내비게이션으로 보이게 하지 않는다.
- 앱 진입 시점부터 **「실시간 전사 텍스트」** / **「요약」** 탭 2개를 항상 노출한다.
- **실시간 전사 텍스트 탭**: 녹음 전·중·후 관계없이 **실시간으로 이어지는 전사 텍스트**를 이 탭 패널에서 본다(녹음 중 갱신 포함).
- **요약 탭**: 아래 **상태**에 따라 다른 UI를 보여 준다(요약 탭 패널 내부).
  - **녹음 전**
  - **녹음 중**
  - **요약 중**
  - **요약 완료**
- 요약 실패·비활성 등은 위 상태 모델에 맞춰 카피·빈 상태·에러 표시를 정한다(제품 문구는 구현 시 확정 가능).
- 요약 생성 API가 아직 없다면, 탭·상태 전환과 플레이스홀더 UI를 먼저 맞추고 이후 API와 연동한다.

## 3. 완료 조건 (Done Criteria)

- [x] 상단 중앙에 Whirr 텍스트가 보이며, 클릭/네비게이션 동작이 없다.
- [x] 모바일에서 좌상단 트리거로 History drawer가 오버레이로 열리고, 라우트 변경 시 닫힌다.
- [x] ≥768px에서 좌측 사이드바 History가 기존 세션 목록과 동일 데이터로 동작한다.
- [x] Record 컨트롤이 탭 영역(전사·요약)보다 위에 고정 배치된다.
- [x] **실시간 전사 텍스트** / **요약** 2탭만 있으며, 탭 줄은 헤더가 아닌 Record 아래 본문에만 있다.
- [x] 실시간 전사 텍스트 탭에 전사 내용이 반영된다.
- [x] 요약 탭에서 녹음 전·녹음 중·요약 중·요약 완료 상태가 구분되어 표시된다.
