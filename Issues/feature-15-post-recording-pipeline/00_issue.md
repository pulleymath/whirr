# Feature 15: 녹음 후 파이프라인·내비게이션·상세 UX

## 제품 결정 (확정)

1. **백그라운드 범위 (A)**: 5분 세그먼트 전사는 기존 `useBatchTranscription` 흐름 유지. **녹음 중지 이후** 마지막 세그먼트 전사 + 요약만 레이아웃/Context 수준에서 실행해, SPA 이동 시에도 완료까지 진행·저장.
2. **세션 저장**: **세션 행을 먼저** IndexedDB에 생성(초기 텍스트/상태)한 뒤, 전사 완료 시 텍스트 갱신, 요약 완료 시 요약 필드 갱신.
3. **`beforeunload`**: **전사/요약 진행 중에도** 경고(탭 닫기·새로고침·외부 이동). 녹음 중에도 기존과 동일하게 필요 시 포함.
4. **SPA 내비게이션**: **녹음 중에만** History 세션 링크 등 내부 이동 제한(설정 버튼과 동일 패턴). 전사/요약 중에는 막지 않음.
5. **동시 녹음**: 백그라운드 파이프라인이 진행 중이면 **새 녹음 시작 비허용**.
6. **클립보드**: **세션 상세 페이지에만** 아이콘 버튼으로 전사 텍스트 복사.
7. **Mock 요약 API**: 자유 구현(지연·고정/간단 응답).
8. **전사 UI**: 세그먼트 전사 **in-flight**일 때만, 현재 전사 텍스트 끝에 혼란이 적은 로딩 표시(예: 말줄임/점 애니메이션).
9. **오디오 다운로드**: **세션 상세만** 유지. **홈 `Recorder`의 오디오 다운로드 버튼 제거**.

## 기술 메모

- `MainAppProviders` 또는 동등 레벨에 파이프라인 Context 추가 검토.
- `fetch`에 `AbortController`로 Provider 언마운트 시 정리 검토.
- `Session` 타입·`db.ts`에 업데이트 API·선택적 요약/상태 필드 검토.
- `Recorder`는 녹음 중지 시 세션 생성 후 파이프라인 enqueue; 실시간/웹스피치 모드는 기존 저장 흐름과 정합성 유지.
