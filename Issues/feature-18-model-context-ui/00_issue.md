# Feature 18: 메인·세션 상세 — 모델 표시, 회의 컨텍스트, 재생성 UX

## 배경

홈에서 녹음 시 사용 중인 스크립트 경로(모드·엔진/배치 모델)와 회의록 작성 모델이 한눈에 보이지 않고, 세션 상세에서는 해당 세션이 어떤 모델로 생성되었는지·회의 컨텍스트가 무엇인지 알기 어렵다. 재생성 시 API에 `glossary`/`sessionContext`가 전달되지 않는 문제도 있다.

## 목표

1. **메인(홈)**
   - 화면을 네 영역으로 구성: ① 녹음 컨트롤 ② 모델(빠른 변경) ③ 회의 컨텍스트 ④ 스크립트 결과.
   - 모델 영역: 전역 `settings`와 동기화되는 드롭다운/선택 UI(스크립트 관련 + 회의록 작성 모델). 녹음 중에는 비활성화(기존 설정 패널과 동일 정책).
   - **레이아웃**: `md` 이상에서는 2열(좌: ①+②, 우: ③+④), 모바일은 세로 스택.

2. **데이터**
   - IndexedDB `Session`에 생성 시점 메타 저장: 스크립트 식별(모드·엔진 또는 배치 모델 id 등), 언어, 회의록 모델 id.
   - DB 버전 업그레이드 및 마이그레이션. **기존 세션**(메타 없음)은 세션 상세에서 **스크립트 모델 표시 블록을 숨김**.

3. **세션 상세**
   - 저장된 스크립트 모델을 **읽기 전용**으로 표시: **모델(또는 엔진) + 모드 라벨 + 언어** 형식.
   - 회의 컨텍스트(참석자·주제·키워드) 표시 및 편집.
   - 세션별 **용어 사전(glossary)** 편집 가능(`MeetingContext`와 일치).
   - 회의록 작성 모델 선택 변경 가능.
   - **재생성** 버튼 하나로: 변경된 컨텍스트·glossary·회의록 모델을 세션에 저장한 뒤 회의록 API 호출·저장. 전역 설정은 변경하지 않음.
   - `fetchMeetingMinutesSummary` 호출 시 `glossary`·`sessionContext` 옵션 반영(버그 수정).

4. **스크립트 모델 재전사**
   - 세션 상세에서는 스크립트 모델 **변경/재전사 없음**, 표시만.

5. **파이프라인**
   - 녹음 후 회의록 완료 시 세션에 `minutesModel` 등 메타가 일관되게 남도록 정리.

## 비범위

- 세션 삭제·검색 등 다른 백로그 항목.
- 스크립트 모델로 오디오 재전사 UI.

## 완료 조건

- 홈에서 4영역 레이아웃 및 모델 빠른 변경이 동작한다.
- 새로 저장되는 세션에 메타가 붙고, 상세에서 읽기 전용 표시·재생성 플로우가 동작한다.
- 레거시 세션은 스크립트 모델 블록이 보이지 않는다.
- 테스트·타입·린트·빌드 통과.
