# Recorder: 단계별 UI·애니메이션

## 배경

홈 `Recorder`에서 녹음 전·중·스크립트 수신 후 정보 계층을 명확히 하고 전환을 부드럽게 한다.

## 요구사항

1. **녹음 시작 전**: `RecordingCard`만 표시한다.
2. **녹음 중**: `SessionContextInput`을 표시한다.
3. **첫 스크립트(세그먼트)가 존재할 때**: `TranscriptView`를 표시한다.
   - 실시간 모드: 최종/부분 텍스트 중 실제 스크립트가 있을 때.
   - 배치 모드: 배치 전사 결과 문자열이 비어 있지 않을 때(파이프라인 표시 텍스트 포함 시 동일 기준 적용).
4. **애니메이션**: `SessionContextInput`·`TranscriptView`가 나타날 때 블러 해제(또는 블러에서 선명으로) 느낌의 등장, `RecordingCard`는 위로 밀리는 레이아웃 전환. 과하지 않고 세련되게.

## 완료 조건

- 위 가시성 규칙이 배치·실시간(Web Speech 포함) 경로에서 일관되게 동작한다.
- 기존 Vitest(Recorder 관련)가 갱신·보강되어 회귀를 막는다.
