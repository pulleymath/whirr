# Feature 8: UI 개선 — History 스타일, 모션, 탭 본문 일관성, sessions 레이아웃

## 1. 개요

Feature 7에서 구성한 메인 UI 셸 위에, **시각적·동작적 품질**을 높이는 후속 작업이다. History 목록은 **ChatGPT 웹앱의 사이드바·대시보드형 대화 목록** 느낌을 참고해 정보 밀도·구분선·호버·타이포를 정리한다. **Drawer·탭 전환** 등 주요 패널에는 **부드러운 애니메이션**을 더해 모바일·데스크톱 모두에서 일관된 느낌을 준다. **실시간 전사 텍스트** 탭과 **요약** 탭의 **본문 영역**은 패딩, 스크롤, 빈 상태·본문 타이포를 맞춰 **한 화면 안의 두 패널**로 읽히게 한다. **`/sessions/[id]`** 등 세션 상세 경로도 **홈과 동일한 헤더·History 사이드바(또는 drawer)** 를 쓰는 **공통 레이아웃**으로 묶어, 앱 전체 내비게이션이 끊기지 않게 한다.

## 2. 상세 기획 (Detailed Plan)

### 2.1 History 리스트 — ChatGPT UI 스타일 참고

- **참고**: ChatGPT 웹 **좌측 사이드바**의 대화 목록(그룹·항목 간 여백, 선택 상태, 호버, 구분)을 **레퍼런스**로 삼는다. 픽셀 단위 복제가 아니라 **정보 구조·시각적 계층**을 맞춘다.
- **방향**:
  - 날짜/그룹 헤더와 항목의 **타이포·색·간격**을 정리해 **대시보드형 목록**으로 읽히게 한다.
  - **현재 세션·선택된 세션** 강조, **호버·포커스** 상태를 키보드·포인터 모두에서 명확히 한다.
  - 모바일 drawer 안의 동일 목록도 **데스크톱 사이드바와 동일 컴포넌트/토큰**을 쓰도록 유지·다듬는다.
- **범위**: 기존 `session-list` 등 데이터 소스·동작은 유지하고, **표현 레이어(마크업·스타일·미세 인터랙션)** 중심으로 변경한다.

### 2.2 Drawer · Tab 등 애니메이션

- **Drawer(모바일 History)**:
  - **열림/닫힘**: 패널 **슬라이드**(또는 동등한 모션) + 필요 시 **배경 딤** 페이드.
  - **라우트 변경 시 닫힘** 동작(Feature 7)은 유지하고, 닫힐 때도 **자연스러운 이징**을 적용한다.
- **탭(실시간 전사 / 요약)**:
  - 활성 탭 전환 시 **인디케이터 또는 콘텐츠 영역**에 짧은 **전환 애니메이션**(예: opacity·슬라이드 중 택1, 과하지 않게).
- **일반 원칙**: `prefers-reduced-motion`을 존중하고, **핵심 기능·접근성**을 해치지 않는 선에서 duration·easing을 통일한다.
- **구현**: 프로젝트에 이미 있는 CSS/애니메이션 유틸·컴포넌트(vaul, tailwind `transition-*` 등)를 우선 사용한다.

### 2.3 실시간 전사 탭 · 요약 탭 본문 UI 일관성

- **목표**: 두 탭의 **콘텐츠 패널**이 같은 **최대 너비·좌우 패딩·상하 여백·스크롤 동작**을 갖도록 맞춘다.
- **타이포**: 본문 텍스트(전사 블록 vs 요약 마크다운/플레이스홀더)의 **기본 글자 크기·줄간·색**을 동일 계열로 통일한다.
- **빈 상태·로딩·에러**: 카드형/중앙 정렬 등 **패턴을 통일**해 탭을 바꿔도 레이아웃이 들쭉날쭉하지 않게 한다.
- **범위**: 헤더·Record 영역·탭 라벨 자체의 스타일 변경은 필요 시 최소로 하되, **본문 영역** 일관성이 우선이다.

### 2.4 sessions 페이지 — 헤더·사이드바 공통 레이아웃

- **대상**: `sessions/[id]`(및 향후 sessions 하위에 추가되는 동일 계열 페이지가 있다면 동일 적용).
- **요구**: 홈(`/`)과 같이 **상단 헤더(Whirr 등)** · **≥768px 좌측 History** · **<768px drawer** 가 보이도록 **레이아웃을 공유**한다.
- **구현 방향**:
  - 공통 **레이아웃 컴포넌트** 또는 **App Router `layout.tsx` 분리**로 중복을 줄이고, 세션 상세 본문만 라우트별로 채운다.
  - 세션 상세에서 History 항목 선택·현재 세션 하이라이트가 **기존 목록과 동일**하게 동작해야 한다.
- **주의**: 세션 페이지 전용으로만 쓰이던 **풀스크린·헤더 없는** 구성이 있다면 제거하거나 레이아웃으로 흡수한다.

## 3. 완료 조건 (Done Criteria)

- [ ] History(사이드바·drawer 동일 목록)가 ChatGPT 스타일 **참고**에 맞게 그룹·항목·선택·호버가 정돈되어 있다.
- [ ] 모바일 History drawer 열림/닫힘(및 딤)에 **의도된 애니메이션**이 있고, `prefers-reduced-motion`이 반영된다.
- [ ] 실시간 전사 / 요약 탭 전환에 **과하지 않은** 모션이 있거나, 팀이 합의한 대안(인디케이터만 등)이 문서화되어 있다.
- [ ] 두 탭 **본문**의 패딩·스크롤·기본 타이포·빈 상태 레이아웃이 **시각적으로 일관**된다.
- [ ] `/sessions/[id]`에서 홈과 동일하게 **헤더 + History(사이드바/drawer)** 가 보이고, 내비게이션이 자연스럽다.
