# `docs/` — 에이전트용 문서 인덱스

작업 전에 태스크 성격에 맞춰 아래 문서에서 맥락을 가져온다. 구현·배포로 바뀌면 관련 문서를 함께 갱신한다.

## 작성 규칙

1. 에이전트가 빠르게 스캔할 수 있게, 문서당 **핵심만** 둔다.
2. 설명이 길어지면 **새 문서를 만들고** 해당 문서에서 링크한다.
3. **예시·파일 경로·코드·프로토콜 필드명** 등 구현 세부는 넣지 않는다. **원리와 경계**만 적는다. (코드 탐색용 **CODEMAP**은 디렉터리 단위만 예외적으로 허용한다.)
