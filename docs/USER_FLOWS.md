# Whirr — 사용자 플로우 (User Flows)

상황별로 **happy path(정상 흐름)**와 **edge case(예외·경계)**를 짧게 정리한다. 화면·상태·내비게이션 관점만 다루고 구현 세부는 [ARCHITECTURE.md](./ARCHITECTURE.md)·[CODEMAP.md](./CODEMAP.md)에 둔다.

읽는 법: 각 절은 **상황 → 정상 흐름 → 예외**. 한 줄 단위로 본다.

---

## 1. 앱 처음 열기

**상황** — 사용자가 루트 URL로 들어온다.

**Happy path**
1. 홈에 **녹음 카드**가 가운데 보인다.
2. **History**가 비어 있다는 안내가 사이드바(또는 드로어)에 뜬다.
3. 헤더의 앱 이름은 항상 홈으로 돌아오는 링크다.

**Edge case**
- **모바일 폭**: History는 숨고, 헤더의 메뉴 버튼으로 드로어에서 연다.
- **이전에 사용한 적이 있는 브라우저**: 같은 브라우저의 로컬 저장소에서 과거 세션을 읽어 History를 채운다. 다른 브라우저·기기는 따로 본다.

---

## 2. 녹음하기 (홈의 메인 루프)

**상황** — 홈에서 회의·메모를 녹음하고 싶다.

**Happy path**
1. (선택) 설정에서 **전역 용어**를 미리 채운다.
2. **시작**을 누른다 → 마이크 권한 허용.
3. 녹음 카드 아래에 **회의 정보·템플릿** 입력 영역이 펼쳐진다.
4. 말한다. 경과 시간·레벨 미터가 움직인다.
5. **종료**를 누른다.
6. 곧이어 자동 처리(§3)가 시작된다.

**Edge case**
- **마이크 거부**: 카드에 오류 안내가 뜨고 녹음이 시작되지 않는다.
- **이전 녹음의 뒷단 처리가 아직 끝나지 않음**: 시작 버튼이 비활성. “이전 녹음을 처리 중입니다” 안내가 카드에 보인다.
- **회의 정보·템플릿 입력 안 함**: 그냥 녹음된다. 회의록 생성 시 맥락만 줄어든다.
- **빈 녹음(아무 발화 없음)**: 종료해도 **새 세션이 만들어지지 않는다**. History는 그대로다.
- **긴 녹음(배치 모드)**: 5분마다 중간 스크립트가 갱신된다는 안내가 나올 수 있다.

---

## 3. 녹음 종료 후 자동 회의록 생성

**상황** — 종료 직후, 사용자는 홈에 있고 **자동으로** 다음이 진행된다.

**Happy path**
1. 카드 아래에 **스크립트 변환 중** 표시.
2. 변환이 끝나면 **회의록 생성 요청**으로 이어진다.
3. 완료되면 **“회의록이 완성되었습니다”** 토스트가 뜬다.
4. 토스트의 **바로 보기**를 누르면 해당 **세션 상세**로 이동한다.

**Edge case**
- **세그먼트 일부 실패**: 카드에 실패 수가 보이고 **재시도** 제어가 나타난다. 재시도 결과로 같은 세션이 갱신된다.
- **회의록 API 실패**: 세션은 **error 상태**가 되고 홈에 한 줄짜리 오류 안내가 뜬다. 세션 상세에서 다시 시도 가능(§7).
- **토스트를 못 누름**: 토스트는 일정 시간 뒤 사라진다. **History에서 같은 세션**을 열면 된다.
- **연속 종료(연달아 두 번 녹음)**: 두 번째 처리는 **대기열**에 쌓이고, 첫 번째가 끝나면 자동으로 진행된다.

---

## 4. History에서 과거 세션 다시 보기

**상황** — 이전에 녹음한 세션을 다시 본다.

**Happy path**
1. History를 연다. 날짜별 그룹과 시간·미리보기가 보인다.
2. 항목을 누르면 **세션 상세**로 이동한다.
3. 현재 보고 있는 세션은 목록에서 강조 표시된다.

**Edge case**
- **녹음 중**: 항목이 비활성. 다른 세션으로 이동할 수 없다.
- **저장된 세션이 없음**: “저장된 세션이 없습니다” 안내가 뜬다.
- **모바일**: 항목을 누르면 드로어가 자동으로 닫히고 본문만 남는다.

---

## 5. 세션 상세 진입

**상황** — `/sessions/{id}` URL로 들어왔다.

**Happy path**
1. **회의록 탭**이 먼저 열린다.
2. 회의록이 있으면 마크다운으로 본다. **전체 복사** 가능.
3. 스크립트는 **스크립트 탭**에서 본다.

**Edge case**
- **로딩 중**: “불러오는 중…”.
- **세션 없음**(URL이 잘못됐거나 다른 브라우저의 id): “세션을 찾을 수 없습니다” + 홈 링크.
- **불러오기 실패**: “세션을 불러오지 못했습니다” + **다시 시도** + 홈 링크.
- **회의록이 아직 없음**: 회의록 탭이 “스크립트 탭에서 생성하세요” 안내로 바뀐다.

---

## 6. 세션 스크립트 편집·저장

**상황** — 자동 생성된 스크립트를 다듬는다.

**Happy path**
1. 스크립트 탭의 큰 텍스트 영역에서 직접 수정한다.
2. **저장**을 누르면 로컬 세션이 갱신된다.
3. **복사** 버튼으로 클립보드에 담을 수 있다.

**Edge case**
- **저장 안 한 채 회의록 생성**: 회의록은 **현재 편집 중인 텍스트** 기준으로 만들어진다. 화면에 “저장하지 않은 내용이 반영됩니다” 안내가 뜬다.
- **저장 실패**: 영역 아래에 빨간 오류 문구.
- **빈 스크립트**: 복사·회의록 생성 모두 비활성.

---

## 7. 세션 회의록 생성·재생성

**상황** — 처음 만들거나, 맥락·템플릿·모델을 바꿔 다시 만든다.

**Happy path**
1. 스크립트 탭 하단의 **회의록 생성** 블록을 연다.
2. 회의 정보·템플릿·세션 전용 용어·회의록 모델을 조정한다.
3. **회의록 생성**(이미 있으면 **재생성**)을 누른다.
4. 끝나면 회의록 탭 내용이 갱신된다.

**Edge case**
- **생성 중**: 입력 필드가 비활성. 버튼은 “생성 중…”.
- **API 실패**: 블록 아래 빨간 오류 문구. 다시 누르면 재시도.
- **스크립트가 비어 있음**: 생성 버튼 비활성.
- **저장 후 화면 새로고침에 실패**: 데이터는 저장됐지만 화면이 못 갱신되었다는 안내. 다른 세션을 열었다 다시 들어오면 보인다.

---

## 8. 오디오 다운로드

**상황** — 로컬에 녹음 세그먼트가 남아 있는 세션이다.

**Happy path**
1. 세션 상세 상단의 **오디오 다운로드**를 누른다.
2. 브라우저의 파일 저장 흐름이 뜬다(ZIP 패키지).

**Edge case**
- **세그먼트가 없는 세션**: 다운로드 버튼이 보이지 않는다.
- **다운로드 중 실패**: 버튼 로딩만 풀린다(별도 토스트 없음). 다시 시도 가능.

---

## 9. 설정 변경

**상황** — 사이드바 하단의 **설정**을 연다.

**Happy path**
1. 모달 패널이 뜬다.
2. **전역 용어 사전**을 한 줄에 하나씩 입력한다.
3. **회의록 모델**을 고른다.
4. 닫기(또는 배경 클릭)로 패널을 닫는다. 변경은 즉시 반영된다.

**Edge case**
- **녹음 중**: 설정 버튼 자체가 비활성. 패널을 열 수 없다.
- **뒷단 처리 중**: 패널은 열리지만 **스크립트·회의록 모델**은 비활성. 그동안 “이전 녹음 처리 중에는 바꿀 수 없습니다” 안내가 보인다.
- **프로덕션 빌드**: 스크립트 모드(실시간/일괄/Web Speech) 선택 UI는 보이지 않는다. 홈은 항상 **녹음 후 일괄**로 동작한다.

---

## 10. 탭 닫기·새로고침 보호

**상황** — 작업 중에 사용자가 실수로 탭을 닫으려 한다.

**Happy path**
1. **녹음 중·전사 중·뒷단 처리 중**에는 브라우저 기본 “나가시겠습니까?” 확인이 뜬다.
2. 모든 처리가 끝나면 이 보호는 자동으로 풀린다.

**Edge case**
- **확인을 무시하고 닫음**: 진행 중이던 처리는 사라진다. 이미 저장된 세션은 남아 있고, 못 끝낸 회의록은 다음에 들어왔을 때 **error 상태**로 보일 수 있다.

---

## 11. (참고) 실시간·Web Speech 모드

**상황** — 개발 빌드에서만 설정으로 모드를 바꿀 수 있다. 프로덕션 사용자에게는 보이지 않는다.

**Happy path(실시간)**
1. 녹음 중 **부분 전사**가 카드 아래로 흘러간다.
2. 종료하면 한 덩어리로 저장되고, 회의록 자동 생성으로 이어진다.

**Edge case**
- **Web Speech 미지원 브라우저**: 옵션이 “(미지원)”으로 비활성. 시작 시 “이 브라우저에서는 사용할 수 없습니다” 안내.
- **Web Speech 중 다른 탭으로 전환**: 다시 돌아오면 자동 재시도 안내가 나올 수 있다.
- **긴 실시간 세션**: “세션이 곧 갱신됩니다” 또는 “스크립트 변환이 중단될 수 있습니다” 안내가 뜰 수 있다.

---

## 12. 개발 전용 프리뷰 화면

**상황** — `/recorder-preview`로 컴포넌트 조합을 본다.

**Happy path** — 개발 빌드에서만 열린다. 컴포넌트 미리보기 용도.

**Edge case** — 프로덕션 빌드에서는 **404**.

---

## 정리: 자주 헷갈리는 경계

| 상황 | 사용자가 기대해야 할 것 |
|------|------------------------|
| 녹음 중 다른 세션 클릭 | 비활성. 끝나야 이동 가능 |
| 녹음 중 설정 클릭 | 비활성 |
| 처리 중 다른 녹음 시작 | 시작 버튼 비활성 |
| 토스트 놓침 | History에서 같은 세션 열기 |
| 스크립트 미저장 + 회의록 생성 | 편집 중 텍스트로 생성됨 |
| 다른 브라우저·기기 | 데이터 따로. 동기화 없음 |
| 오디오 다운로드 버튼이 없는 세션 | 로컬 세그먼트가 없음 |
