# Whirr — 사용자 플로우 (User Flows)

상황별로 **happy path(정상 흐름)**와 **edge case(예외·경계)**를 짧게 정리한다. 화면·상태·내비게이션 관점만 다루고 구현 세부는 [ARCHITECTURE.md](./ARCHITECTURE.md)·[CODEMAP.md](./CODEMAP.md)에 둔다.

읽는 법: 각 절은 **상황 → 정상 흐름 → 예외**. 한 줄 단위로 본다.

---

## 1. 앱 처음 열기

**상황** — 사용자가 루트 URL로 들어온다.

**Happy path**

1. 홈에 **노트 작업면**(편집 가능한 제목, 참석자·주제·키워드, **요약 형식** 드롭다운, **AI 요약** / **스크립트** 탭)이 보이고, **녹음 카드**는 본문 스크롤과 무관하게 화면 **최하단**에 붙어 있다. 본문은 여백을 포함해 스크롤되며, 기본 탭은 **AI 요약**(선택한 요약 형식의 출력 구조 미리보기 또는 직접입력 형식 지침)이다.
2. **History**가 비어 있다는 안내가 사이드바(또는 드로어)에 뜬다.
3. 헤더의 앱 이름은 항상 홈으로 돌아오는 링크다.

**Edge case**

- **모바일 폭**: History는 숨고, 헤더의 메뉴 버튼으로 드로어에서 연다.
- **이전에 사용한 적이 있는 브라우저**: 같은 브라우저의 로컬 저장소에서 과거 세션을 읽어 History를 채운다. 다른 브라우저·기기는 따로 본다.

---

## 2. 녹음하기 (홈의 메인 루프)

**상황** — 홈에서 회의·메모를 녹음하고 싶다.

**Happy path**

1. (선택) 설정에서 **전역 용어**를 미리 채운다.
2. (선택) **노트 제목**(저장 시 세션에 함께 저장되며 History에 우선 표시. 비워 두면 **「새로운 노트」**로 저장)과 **참석자**(쉼표로 여러 명), **주제·키워드**, **요약 형식**(드롭다운)을 미리 채운다. **직접입력**을 고르면 **AI 요약** 탭에서 형식 지침을 편집한다. idle 상태부터 노출되며, 제목·형식을 비워 두어도 녹음 시작에는 영향이 없다.
3. **시작**을 누른다 → 마이크 권한 허용.
4. 말한다. **녹음 카드**에서 경과 시간·레벨 미터가 움직인다. 제목·회의 필드·템플릿은 아래 작업면에 계속 보인다. 스크립트는 **스크립트** 탭에서 확인한다(녹음 전에도 탭을 열면 빈 상태 안내가 보인다).
5. **종료**를 누른다.
6. 곧이어 자동 처리(§3)가 시작되고, 회의 정보·요약 형식은 다음 녹음을 위해 초기값으로 되돌려진다.

**Edge case**

- **마이크 거부**: 녹음 카드에 오류 안내가 뜨고 녹음이 시작되지 않는다.
- **이전 녹음의 뒷단 처리가 아직 끝나지 않음**: 시작 버튼이 비활성. “이전 녹음을 처리 중입니다” 안내가 **녹음 카드**에 한 번 보인다. 회의 필드·템플릿은 보이지만 입력은 비활성이며 “요약 생성 중에는 수정할 수 없습니다.” 안내가 함께 뜬다.
- **회의 정보·템플릿 입력 안 함**: 그냥 녹음된다. 요약 생성 시 맥락만 줄어든다.
- **세션 저장 실패**: enqueue까지 가지 못한 경우 입력값(노트 제목·참석자·주제·키워드·요약 형식)이 **그대로 유지**되어 사용자가 다시 시도할 수 있다. 작업면 **아래**에 “세션을 저장하지 못했습니다.” 오류 문구가 뜬다.
- **빈 녹음**: 의미 있는 녹음 내용이 없으면 새 세션을 만들지 않는다. History는 그대로이고, 회의 정보·템플릿 입력값은 유지된다.
- **배치 모드 녹음 중 안내**: 녹음이 진행 중이면 녹음 카드·스크립트 영역에 “3분마다 스크립트 결과가 업데이트됩니다” 안내가 **항상** 보인다(중간 스크립트는 최대 약 3분 주기로 갱신될 수 있다).

## 3. 녹음 종료 후 자동 요약 생성

**상황** — 종료 직후, 사용자는 홈에 있고 **자동으로** 다음이 진행된다.

**Happy path**

1. 작업면·녹음 카드 근처에 **스크립트 변환 중** 표시.
2. 변환이 끝나면 **요약 생성 요청**으로 이어진다.
3. 완료되면 **“요약이 완성되었습니다”** 토스트가 뜬다.
4. 토스트의 **바로 보기**를 누르면 해당 **세션 상세**로 이동한다.

**Edge case**

- **세그먼트 일부 실패**: 녹음 카드에 실패 수가 보이고 **재시도** 제어가 나타난다. 재시도 결과로 같은 세션이 갱신된다.
- **요약 API 실패**: 세션은 **error 상태**가 되고 홈에 한 줄짜리 오류 안내가 뜬다. 세션 상세에서 다시 시도 가능(§7).
- **토스트를 못 누름**: 성공 토스트는 **8초** 후 자동으로 사라진다. **History에서 같은 세션**을 열면 된다.
- **연속 완성 토스트**: 짧은 간격으로 또 완성되면 이전 성공 알림이 같은 자리에서 교체될 수 있다.
- **연속 종료(연달아 두 번 녹음)**: 뒷단 처리가 진행 중일 때 새 작업은 순서대로 대기한다. 같은 세션의 중복 작업은 한 번만 처리된다.

---

## 4. History에서 과거 세션 다시 보기

**상황** — 이전에 녹음한 세션을 다시 본다.

**Happy path**

1. History를 연다. 날짜별 그룹과 시간·한 줄 표시가 보인다. 세션에 **노트 제목**이 있으면 스크립트 미리보기 대신 **제목**이 먼저 보이고, 제목이 없는 예전 세션은 기존처럼 스크립트 미리보기로 본다.
2. 항목을 누르면 **세션 상세**로 이동한다.
3. 현재 보고 있는 세션은 목록에서 강조 표시된다.

**Edge case**

- **녹음 중**: 항목이 비활성. 다른 세션으로 이동할 수 없다.
- **저장된 세션이 없음**: “저장된 세션이 없습니다” 안내가 뜬다.
- **모바일**: 항목을 누르면 드로어가 자동으로 닫히고 본문만 남는다.

---

## 5. 세션 상세 진입

**상황** — `/sessions/{id}` URL로 들어왔다.

**Happy path**

1. 기본으로 **AI 요약** 탭이 열린다.
2. 상단에는 노트 제목과, 세션에 오디오가 있으면 **오디오 다운로드**, **편집**(모달)이 한 줄에 보인다.
3. 요약이 있으면 마크다운으로 본다. **전체 복사** 가능.
4. **스크립트** 탭에서는 위쪽 한 줄에 전사 메타(있을 때)·**복사**가 있고, 아래에 스크립트 본문이 이어진다(읽기 전용).

**Edge case**

- **로딩 중**: “불러오는 중…”.
- **세션 없음**(URL이 잘못됐거나 다른 브라우저의 id): “세션을 찾을 수 없습니다” + 홈 링크.
- **깨진 세션 URL**: “세션을 찾을 수 없습니다” + 홈 링크로 본다.
- **불러오기 실패**: “세션을 불러오지 못했습니다” + **다시 시도** + 홈 링크.
- **요약이 아직 없음**: 요약 탭에 안내 문구가 보이고, **편집**에서 스크립트·맥락을 맞춘 뒤 재생성할 수 있다.

---

## 6. 세션 스크립트·맥락 편집·저장

**상황** — 저장된 스크립트나 회의 맥락·요약 형식을 고친다.

**Happy path**

1. 상단 **편집**을 눌러 모달을 연다.
2. 모달 안 **AI 요약**(읽기 전용)·**요약 형식 미리보기**·**스크립트** 탭으로 이동해 필요한 항목을 수정한다. 요약 형식만 바꾸면 미리보기 탭으로 맞춰 줄 수 있다.
3. **스크립트** 탭에서 본문을 고친다.
4. **저장**으로 로컬 세션을 갱신한다(성공 시 모달이 닫힌다).

**보조** — 상세 화면 **스크립트** 탭 상단의 **복사**는 모달을 열지 않고도 쓸 수 있다.

**Edge case**

- **닫기·배경 클릭·Esc**: 저장하지 않은 변경이 있으면 확인 후 닫힌다.
- **저장 실패**: 모달 본문 아래에 빨간 오류 문구.
- **빈 스크립트**: 모달에서 **요약 재생성** 계열 버튼이 비활성일 수 있다. 복사도 비활성일 수 있다.

---

## 7. 세션 요약 생성·재생성

**상황** — 처음 만들거나, 맥락·형식·모델·스크립트를 바꿔 다시 만든다.

**Happy path**

1. **편집**으로 모달을 연다.
2. 회의 맥락·요약 형식·작성 모델·스크립트 등을 조정한다.
3. **현재 세션에 요약 재생성** 또는 **새 세션에 요약 재생성**을 고른다. **현재 세션**이면 기존 요약 덮어쓰기 확인이 먼저 뜬다.
4. 처리가 시작되면 모달은 닫히고, 상세 화면에 진행 안내가 보인다. 끝나면 **AI 요약** 탭 내용이 갱신된다. **새 세션**이면 새 노트로 이어지는 안내·링크가 보일 수 있다.

**Edge case**

- **생성 중**: 요약 파이프라인이 바쁠 때 입력·편집은 비활성에 가깝게 막힌다.
- **API 실패**: 상세에 오류 안내. 같은 흐름으로 다시 시도할 수 있다.
- **레이트 리밋(429)**: “요청이 많아 잠시 후 다시 시도해 주세요.” 안내가 우선된다.
- **스크립트가 비어 있음**: 재생성 버튼이 비활성.
- **저장 후 화면 새로고침에 실패**: 데이터는 저장됐지만 화면이 못 갱신되었다는 안내. 다른 세션을 열었다 다시 들어오면 보인다.

---

## 8. 오디오 다운로드

**상황** — 로컬에 녹음 세그먼트가 남아 있는 세션이다.

**Happy path**

1. 세션 상세 상단의 **오디오 다운로드**를 누른다.
2. 브라우저의 파일 저장 흐름이 뜬다(ZIP 패키지).

**Edge case**

- **세그먼트가 없는 세션**: 다운로드 버튼이 보이지 않는다.
- **다운로드 중 실패**: 버튼 로딩만 풀린다(별도 토스트 없음). 다시 시도 가능.

---

## 9. 설정 변경

**상황** — 사이드바 하단의 **설정**을 연다.

**Happy path**

1. 모달 패널이 뜬다.
2. **전역 용어 사전**을 한 줄에 하나씩 입력한다.
3. **요약 모델**을 고른다.
4. 닫기(또는 배경 클릭)로 패널을 닫는다. 변경은 즉시 반영된다.

**Edge case**

- **녹음 중**: 설정 버튼 자체가 비활성. 패널을 열 수 없다.
- **뒷단 처리 중**: 패널은 열리지만 **스크립트·요약 모델**은 비활성. 그동안 “이전 녹음 처리 중에는 바꿀 수 없습니다” 안내가 보인다.
- **프로덕션 빌드**: 스크립트 모드(실시간/일괄/Web Speech) 선택 UI는 보이지 않는다. 홈은 항상 **녹음 후 일괄**로 동작한다.

## 10. 탭 닫기·새로고침 보호

**상황** — 작업 중에 사용자가 실수로 탭을 닫으려 한다.

**Happy path**

1. 아래 조건 **어느 하나라도 참**이면 브라우저 기본 “나가시겠습니까?” 확인이 뜬다: **녹음 중** · **배치 모드에서 스크립트 일괄 전사 중** · **뒷단 처리 중**(스크립트 변환·요약 생성 파이프라인이 바쁜 상태) · **세션 상세에서 요약 생성 요청이 서버로 진행 중**(버튼이 “생성 중…”인 동안).
2. 위 조건이 모두 거짓이면 보호는 자동으로 풀린다.

**Edge case**

- **확인을 무시하고 닫음**: 진행 중이던 처리는 사라진다. 이미 저장된 세션은 남아 있고, 못 끝낸 요약은 다음에 들어왔을 때 **error 상태**로 보일 수 있다.

## 11. (참고) 실시간·Web Speech 모드

**상황** — 개발 빌드에서만 설정으로 모드를 바꿀 수 있다. 프로덕션 사용자에게는 보이지 않는다.

**Happy path(실시간)**

1. 녹음 중 **부분 전사**가 작업면 스크립트 영역으로 흘러간다.
2. 종료하면 한 덩어리로 저장되고, 요약 자동 생성으로 이어진다.

**Edge case**

- **Web Speech 미지원 브라우저**: 옵션이 “(미지원)”으로 비활성. 시작 시 “이 브라우저에서는 사용할 수 없습니다” 안내.
- **Web Speech 중 다른 탭으로 전환**: 사용자가 앱 탭으로 돌아오면 엔진이 **한 번** 자동 재시작을 시도한다. 두 번째 이후 같은 복귀에서는 추가 자동 재시작을 하지 않는다(녹음 카드에 별도 “재시도 안내” 문구가 항상 뜨지는 않을 수 있다).
- **긴 스트리밍 녹음(실시간·Web Speech)**: 경과 시간이 **약 55분**에 도달하면 녹음 카드에 경고 톤으로 안내가 뜬다. **OpenAI 실시간** 엔진: “세션이 곧 갱신됩니다.” **Web Speech API** 모드: “녹음 시간이 길어지고 있습니다. 스크립트 변환이 중단될 수 있습니다.”
- **실시간(OpenAI) 연결 유지 재시도**: 선제 재연결 등으로 복구가 진행될 때 녹음 카드에 “세션을 곧 갱신합니다.”가 **잠깐** 보였다가 **약 4초** 안에 사라질 수 있다.

## 12. 개발 전용 프리뷰 화면

**상황** — `/recorder-preview`로 컴포넌트 조합을 본다.

**Happy path** — 개발 빌드에서만 열린다. 홈과 동일한 **녹음 카드 + 노트 작업면**(요약 형식 드롭다운, AI 요약/스크립트 탭 포함) 조합을 단계 버튼으로 미리 본다.

**Edge case** — 프로덕션 빌드에서는 **404**.

---

## 13. 로컬 저장 실패

**상황** — 브라우저 IndexedDB 쓰기가 실패한다(저장 공간·잠금·환경 오류 등).

**Happy path** — 발생하지 않는다.

**Edge case**

- **녹음 종료 직후 세션 저장 실패**: 홈 작업면 하단에 “세션을 저장하지 못했습니다.”가 보이고, 자동 요약 파이프라인은 이어가지 않는다.
- **세션 상세에서 스크립트 저장 실패**: 편집 모달 또는 상세 본문 아래에 빨간 오류 문구. **저장**으로 다시 시도한다.
- **세션 상세에서 요약 재생성·저장 실패**: 상세 본문에 오류 안내. **편집**을 다시 열어 재시도할 수 있다.

## 정리: 자주 헷갈리는 경계

| 상황                             | 사용자가 기대해야 할 것     |
| -------------------------------- | --------------------------- |
| 녹음 중 다른 세션 클릭           | 비활성. 끝나야 이동 가능    |
| 녹음 중 설정 클릭                | 비활성                      |
| 처리 중 다른 녹음 시작           | 시작 버튼 비활성            |
| 토스트 놓침                      | History에서 같은 세션 열기  |
| 스크립트 미저장 상태에서 재생성 | 모달에 입력된 값이 파이프라인에 반영되는 흐름을 따른다 |
| 다른 브라우저·기기               | 데이터 따로. 동기화 없음    |
| 오디오 다운로드 버튼이 없는 세션 | 로컬 세그먼트가 없음        |
| IndexedDB 저장 실패              | 홈·세션 상세 오류 문구, §13 |
