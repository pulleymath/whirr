# Whirr — 아키텍처(개념)

PRD의 요구를 만족시키기 위한 **경계·책임·데이터 위치**만 정의한다. 구현 세부는 코드베이스를 본다.

## 신뢰·실행 경계

- **브라우저**: UI, 마이크 캡처, STT와의 실시간 채널, 로컬 세션 저장·조회.
- **호스팅 앱(예: Next.js + Vercel)**: 장기 비밀을 알지 못하게 하기 위한 **짧은 수명 자격 증명 발급** 등 최소 서버 역할만 담당.
- **외부 STT**: 오디오 수신·전사·결과 푸시. 공급자 정책 하에 처리.

별도의 범용 백엔드(오디오 중계·비즈니스 API)는 두지 않는다.

## 구성 요소 책임

| 영역            | 책임                                                                   |
| --------------- | ---------------------------------------------------------------------- |
| 녹음 파이프라인 | 캡처된 오디오를 STT가 요구하는 형식·주기 규칙에 맞게 공급              |
| STT 어댑터 계층 | 공급자별 연결·프로토콜을 **동일한 UI 이벤트**(부분/최종/오류)로 정규화 |
| 전사 UI         | 부분 문장 갱신, 확정 문장 누적 표시                                    |
| 로컬 저장소     | 세션 단위로 시각·전사 텍스트만 보관·인덱싱(목록 정렬·날짜 그룹)        |

## 데이터가 머무는 곳

- **유지**: 전사 텍스트(클라이언트 DB).
- **유지하지 않음**: 녹음 오디오 원본(앱·서버 장기 보관 대상 아님).
- **서버 영속**: MVP에서 사용자 전사 본문을 저장하지 않음.

## 교체 가능성

STT 공급자·모델이 바뀌어도 **UI는 어댑터 인터페이스**만 보도록 한다. 자체 호스팅 STT로 바꿀 경우 브라우저가 붙는 끝점과 어댑터 구현만 늘어날 수 있으며, 제품 요구는 [PRD.md](./PRD.md)를 따른다.

## 운영·비밀

- 장기 키는 서버 환경에만 둔다.
- 공개 번들에 비밀이 섞이지 않게 한다.
- 토큰 발급 경로는 남용 완화를 위한 **속도 제한 등 정책**을 둘 수 있다(상세는 [DECISIONS.md](./DECISIONS.md)).
