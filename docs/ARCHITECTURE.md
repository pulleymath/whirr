# Whirr — 아키텍처(개념)

PRD의 요구를 만족시키기 위한 **경계·책임·데이터 위치**만 정의한다. 구현 세부는 코드베이스를 본다.

## 신뢰·실행 경계

- **브라우저**: UI, 마이크 캡처, STT와의 실시간 채널, 로컬 세션 저장·조회.
- **호스팅 앱(예: Next.js + Vercel)**: 장기 비밀을 알지 못하게 하기 위한 **짧은 수명 자격 증명 발급** 등 최소 서버 역할만 담당.
- **외부 STT**: 오디오 수신·전사·결과 푸시. 공급자 정책 하에 처리.

별도의 범용 백엔드(오디오 중계·비즈니스 API)는 두지 않는다.

## 일괄 전사(녹음 후) 경로

- **브라우저**: 마이크 → `MediaRecorder`로 압축 Blob(임시) → `FormData`로 앱 API에 업로드.
- **호스팅 앱**: `POST /api/stt/transcribe`가 크기·MIME·모델·레이트 리밋을 검증한 뒤 OpenAI **HTTP 전사**로 프록시하고, **텍스트만** 클라이언트에 돌려준다.
- **UI**: 실시간 WSS 어댑터(`useTranscription`)와는 별도 훅이지만, 전사 표시·세션 저장 흐름은 동일 화면에서 이어진다.

## 녹음 후 파이프라인(마지막 세그먼트·회의록)

배치 녹음이 끝난 뒤 **5분 단위로 이미 전사된 본문**에 더해, **마지막 오디오 조각**만 클라이언트에서 `POST /api/stt/transcribe`로 전사하고, 합쳐진 전문을 `POST /api/meeting-minutes`로 보낸다. 구간 전사가 실패하면 **세그먼트 회전 시점과 브라우저 `online` 복구 시** 동일 큐에서 시간순으로 재시도하고, 모든 구간이 성공한 뒤에만 세션 저장과 회의록 단계로 이어진다(수동 재시도 포함). 서버는 전사 본문을 **청크 map → 단일 reduce**로 합쳐 OpenAI Chat Completions로 **회의록**을 생성하고, 응답 본문은 기존과 같이 `{ summary: string }` 형태로 돌아와 로컬 세션의 **`summary`** 필드에 저장된다(UI에서는 "회의록"으로 표시). 이 비동기 단계는 **`PostRecordingPipelineProvider`**(메인 레이아웃)에서 돌아가며, SPA 내비게이션 후에도 같은 탭에서는 완료까지 이어진다. 세션 레코드에는 진행 상태를 위해 **`status`**(예: transcribing, summarizing, ready, error)가 사용된다.

## 구성 요소 책임

| 영역            | 책임                                                                                                                                                                                                                                                                                                                 |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 녹음 파이프라인 | 캡처된 오디오를 STT가 요구하는 형식·주기 규칙에 맞게 공급                                                                                                                                                                                                                                                            |
| STT 어댑터 계층 | 공급자별 연결·프로토콜을 **동일한 UI 이벤트**(부분/최종/오류)로 정규화. **Web Speech API**는 브라우저 내장 인식기를 쓰며 토큰 없이 연결(`tokenlessProvider` 경로)한다. **장시간 세션**에서는 프로바이더가 끊김·만료를 알리고, **토큰 기반 실시간 경로**는 훅에서 **재연결·카운터·사용자 문구**를 오케스트레이션한다. |
| 전사 UI         | 부분 문장 갱신, 확정 문장 누적 표시                                                                                                                                                                                                                                                                                  |
| 로컬 저장소     | 세션 단위로 시각·전사 텍스트만 보관·인덱싱(목록 정렬·날짜 그룹)                                                                                                                                                                                                                                                      |
| 앱 설정(클라)   | 전사 모드·엔진·언어 등 **브라우저 전용** 설정: Context + `localStorage`                                                                                                                                                                                                                                              |
| 메인 레이아웃   | `(main)` 트리를 **`MainAppProviders`**로 감싸 전역 Context를 주입한다.                                                                                                                                                                                                                                               |

## 데이터가 머무는 곳

- **유지**: 전사 텍스트(클라이언트 DB).
- **지역 한정(일시)**: 배치 세션에 한해 오디오 세그먼트가 **사용자 기기 IndexedDB**에만 남을 수 있다(서버로 업로드하지 않음). 세션 상세의 zip 다운로드는 **브라우저에서만** 생성·저장된다.
- **유지하지 않음**: 녹음 오디오 원본(앱·서버 장기 보관 대상 아님).
- **서버 영속**: MVP에서 사용자 전사 본문을 저장하지 않음.
- **브라우저만**: 전사 관련 **사용자 설정**(모드·엔진·모델·언어)은 `localStorage`에만 두고, 비밀값은 넣지 않는다.

## 교체 가능성

STT 공급자·모델이 바뀌어도 **UI는 어댑터 인터페이스**만 보도록 한다. 자체 호스팅 STT로 바꿀 경우 브라우저가 붙는 끝점과 어댑터 구현만 늘어날 수 있으며, 제품 요구는 [PRD.md](./PRD.md)를 따른다.

## 운영·비밀

- 장기 키는 서버 환경에만 둔다.
- 공개 번들에 비밀이 섞이지 않게 한다.
- 토큰 발급 경로는 남용 완화를 위한 **속도 제한 등 정책**을 둘 수 있다(상세는 [DECISIONS.md](./DECISIONS.md)).
