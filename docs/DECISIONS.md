## (참고) 실시간 STT 세션 복원력

- **OpenAI Realtime**: 업스트림 **세션당 약 60분** 한도에 맞춰 **연결 기준 약 55분**에 선제로 정상 종료 후, 클라이언트가 **새 토큰으로 재연결**한다. 동일 녹음 세션에서 **자동 재연결 시도는 최대 3회**까지 하며, 초과 시 사용자에게 **수동으로 녹음 재시작**을 안내한다.
- **AssemblyAI·기타 실시간 경로**: 비정상 종료는 사용자에게 알리고, 토큰 경로에서는 위와 동일한 재연결 정책을 탄다(공급자별 선제 타이머는 OpenAI에 한정).
- **녹음 후 일괄 전사**: 전사 실패는 자동 재시도 후 사용자가 다시 시도할 수 있게 한다. 클라이언트는 재시도에 필요한 임시 오디오만 보관한다.

## STT 경로

- **기본 제품 경로**: 녹음 후 일괄 전사. 실시간 미리보기보다 안정적인 저장·요약 흐름을 우선한다.
- **대체 경로**: 실시간 STT와 Web Speech 계열은 개발·교체 가능성 범위로 유지한다.
- **원칙**: 공급자별 프로토콜·샘플레이트 차이는 **Provider(어댑터)** 안에서 흡수한다.

## 프론트엔드·앱 형태

- **Next.js(App Router)**: UI·라우팅·최소 API를 한 리포지토리에서 다룬다.
- **Provider 추상화**: STT 구현체 교체 시 UI 변경을 최소화한다.

## 백엔드 범위

- **별도 백엔드 서버 없음**: 오디오 중계용 장기 서버를 두지 않는다.
- **예외적 서버 로직**: 비밀 보호·짧은 토큰 발급 등, 브라우저만으로 불가능한 최소만 호스팅 함수로 처리한다.

## 배포

- **Vercel**(또는 동급): 프론트와 API 라우트를 함께 배포한다.

## 서버 요청 보호

- 외부 AI 서비스와 연결되는 서버 요청에는 **클라이언트 식별 기반 속도 제한**을 둔다. 트래픽이 늘면 중앙 저장소·엣지 정책 등으로 이전 검토.

## UI 오류 표시

- 업스트림 원문 오류를 그대로 노출하지 않고, **사용자용 문구로 정규화**한다.

## 전사·요약 설정(클라이언트)

- **상태**: 전사·요약 모델, 언어, 개발용 전사 모드는 브라우저 전용 설정으로 보관한다.
- **렌더링**: 브라우저 저장소 읽기는 클라이언트 마운트 이후에만 수행해 초기 렌더 불일치를 피한다.
- **일괄 전사**: 오디오 원본은 앱·서버에 장기 보관하지 않으며, 긴 녹음에는 사용자 안내와 자동 종료 정책을 둔다.
- **Web Speech**: 브라우저 내장 전사는 별도 키 없이 동작하지만, 품질과 지원 범위가 환경에 따라 달라 개발·대체 경로로 본다.
- **원칙**: 클라이언트 저장값은 신뢰하지 않고, 서버 경계에서 허용 목록으로 다시 검증한다.

## 과거 검토(요지)

초기에 AssemblyAI 등 **브라우저 실시간 연결·비용**을 기준으로 후보를 비교했고, 안정성·요구사항에 따라 현재 경로를 정했다. 세부 비교표는 유지보수 부담이 커서 문서에서는 생략한다.
