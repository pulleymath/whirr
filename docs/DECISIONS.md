# Whirr — 기술 결정

근거가 길어지면 이 문서에 요지만 남기고, 장문 분석이 필요하면 별도 노트를 만들어 [README.md](./README.md) 목차에 링크한다.

## 실시간 STT 세션 복원력(resilience)

- **OpenAI Realtime**: 업스트림 **세션당 약 60분** 한도에 맞춰 **연결 기준 약 55분**에 선제로 정상 종료 후, 클라이언트가 **새 토큰으로 재연결**한다. 동일 녹음 세션에서 **자동 재연결 시도는 최대 3회**까지 하며, 초과 시 사용자에게 **수동으로 녹음 재시작**을 안내한다.
- **AssemblyAI·기타 WSS**: 비정상 **WebSocket 종료**는 사용자에게 알리고, 토큰 경로에서는 위와 동일한 **훅 단 재연결** 정책을 탄다(공급자별 선제 타이머는 OpenAI에 한정).
- **일괄 전사(batch)**: 업로드용 **Blob은 전사 성공·새 녹음 시작 전까지 유지**한다. **5xx·네트워크 실패**는 **2·4초 백오프**로 최대 **2회 자동 재시도**(총 3회 시도), **4xx**는 재시도하지 않는다. UI에서 **수동 «다시 시도»**를 제공한다.

## STT 엔진(현재 기본)

- **기본**: OpenAI Realtime **전사(transcription)** 경로. 브라우저가 **단기 자격 증명**으로 WSS에 직접 연결한다.
- **레거시**: AssemblyAI 기반 경로는 코드에 남겨 두되, 기본 제품 경로는 OpenAI 쪽이다.
- **원칙**: 공급자별 프로토콜·샘플레이트 차이는 **Provider(어댑터)** 안에서 흡수한다.

## 프론트엔드·앱 형태

- **Next.js(App Router)**: UI·라우팅·최소 API를 한 리포지토리에서 다룬다.
- **Provider 추상화**: STT 구현체 교체 시 UI 변경을 최소화한다.

## 백엔드 범위

- **별도 백엔드 서버 없음**: 오디오 중계용 장기 서버를 두지 않는다.
- **예외적 서버 로직**: 비밀 보호·짧은 토큰 발급 등, 브라우저만으로 불가능한 최소만 호스팅 함수로 처리한다.

## 배포

- **Vercel**(또는 동급): 프론트와 API 라우트를 함께 배포한다.

## 토큰 API 보호

- 발급 엔드포인트에 **클라이언트 식별 기반 속도 제한**을 둔다. 서버리스 다인스턴스에서는 전역 한계가 아닐 수 있음 — 트래픽 증가 시 중앙 저장소·엣지 정책 등으로 이전 검토.

## UI 오류 표시

- 업스트림 원문 오류를 그대로 노출하지 않고, **사용자용 문구로 정규화**한다.

## 전사 설정(클라이언트)

- **상태**: 전사 모드·실시간 엔진·일괄 모델·언어는 **React Context**(`SettingsProvider`)로 보관하고, 브라우저 **`localStorage`** 키 `whirr:transcription-settings`에 JSON으로 영속화한다.
- **SSR**: 저장소 읽기는 **마운트 이후**에만 수행해 서버 렌더와 하이드레이션 오류를 피한다.
- **일괄 전사(batch)**: 브라우저 `MediaRecorder`로 webm/opus 등 **임시 Blob**만 메모리에 둔 뒤 `POST /api/stt/transcribe`가 OpenAI **REST 전사**로 프록시한다. 오디오 원본은 앱·서버에 장기 보관하지 않으며, 업로드 한도에 맞춰 **약 1시간**을 소프트(55분)·하드(60분) 타이머로 안내·자동 종료한다. **서버는 허용 모델 ID 화이트리스트**로만 업스트림을 호출한다.
- **`webSpeechApi`**: 브라우저 **Web Speech API**(`SpeechRecognition`)로 전사한다. API 키·토큰 없이 동작하나, 브라우저·OS에 따라 품질이 들쭉날쭉하고(Chrome은 클라우드 전사 경유 가능) 외부 STT 대비 기능이 제한된다. `TranscriptionProvider`와 동일한 부분/최종 콜백으로 UI에 붙인다.
- **원칙**: 저장값(`batchModel` 등)이 임의 문자열이 될 수 있으므로, **서버 API에 넘긴 뒤**에도 허용 목록으로 다시 검증한다.

## 과거 검토(요지)

초기에 AssemblyAI 등 **브라우저 WSS·비용**을 기준으로 후보를 비교했고, 안정성·요구사항에 따라 기본 경로를 OpenAI 전사로 옮겼다. 세부 비교표는 유지보수 부담이 커서 문서에서는 생략한다.
