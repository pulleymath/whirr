# Whirr — 디자인 토큰

이 문서는 **시각 토큰의 의도와 경계**만 둔다. 패턴·상태·모션·접근성은 [UI_PATTERNS.md](./UI_PATTERNS.md). 채택 이유·대안은 [DECISIONS.md](./DECISIONS.md). 코드 컨벤션은 [CONVENTIONS.md](./CONVENTIONS.md).

읽는 법: 각 절은 **무엇을 / 경계(금지·예외)** 순.

---

## 1. 톤·무드

- 미니멀·정돈된 도구. 사용자의 집중을 방해하지 않는 시각 언어.
- 강한 색은 **의미가 있을 때만** 등장한다. 장식적 색·그라디언트는 쓰지 않는다.
- **다크모드 단일** 지원. 라이트 모드는 비목표.

---

## 2. 색의 의미

강한 색은 **두 자리만** 둔다. 두 자리의 의미는 절대 겹치지 않는다.

| 색      | 역할                                                                          |
| ------- | ----------------------------------------------------------------------------- |
| `rose`  | **녹음 행위 전용** — 시작·정지·녹음 중 인디케이터. 다른 어디에도 쓰지 않는다. |
| `sky`   | **시스템 액센트** — 포커스, 처리 진행률, 활성 강조를 모두 이 한 색으로 통일.  |
| `amber` | 경고. 텍스트·아이콘 톤으로만 쓰고, 배경 면적으로 강조하지 않는다.             |
| `red`   | 오류. 텍스트 위주. 녹음 `rose`와 hue를 구분해 의미 충돌을 막는다.             |
| `zinc`  | 그 외 모든 면·테두리·텍스트 위계의 베이스.                                    |

**경계**

- 새 의미 색을 임의로 추가하지 않는다. 새 의미가 필요하면 먼저 위 5색의 명도 단계로 표현 가능한지 본다.
- `rose`는 "이 색이 보이면 녹음이다"라는 시각적 약속이다. 강조·치명도 표시 등 다른 의도에 차용하지 않는다.

---

## 3. 타이포

- 본문은 산세리프 가변폭이 기본이다.
- **모노스페이스 + 표 숫자(tabular-nums)** 는 고정폭이 의미 있는 자리에만 쓴다 — 시간, 진행 수치, 모델·식별자.
- 같은 화면의 위계는 **세 단계 이내**(본문·보조·매우 보조)로 유지한다. 굵기는 보통/세미볼드까지.
- heading 크기로 위계를 만들지 않는다. 의미 단위(섹션·카드)로 위계를 만든다.

---

## 4. 간격·라운드

라운드는 **의미 단계**로 사용한다. 한 화면에서 단계를 섞어 시각적 잡음을 만들지 않는다.

| 단계             | 용도                        |
| ---------------- | --------------------------- |
| 가장 큰 라운드   | 작업 단위 카드(섹션)        |
| 중간 라운드      | 버튼·입력·텍스트 영역       |
| 원형             | 인디케이터·미터·진행률 트랙 |
| 가장 작은 라운드 | 정지·중단 같은 단정한 표시  |

- 카드 패딩과 컨텐츠 간격은 **각 한 단계만** 쓴다. 공간을 미세하게 다르게 두지 않는다.
