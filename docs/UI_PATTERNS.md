# Whirr — UI 패턴

새 화면·컴포넌트를 만들 때 **무엇을 고르고 무엇을 피할지**의 경계만 둔다. 색·라운드·타이포는 [DESIGN.md](./DESIGN.md). 사용자 흐름은 [USER_FLOWS.md](./USER_FLOWS.md).

읽는 법: 각 절은 **언제 / 무엇을 / 경계(금지)** 순.

---

## 1. 컴포넌트 선택

작업 단위는 **카드**로 묶는다 — 녹음·스크립트·세션 상세 같은 한 가지 일이 한 카드다.

액션 컴포넌트는 다음 4종이 기본이다.

| 종류              | 언제                                                                      |
| ----------------- | ------------------------------------------------------------------------- |
| 일반 버튼(텍스트) | 본문 액션. 강도에 따라 primary/outline/ghost 변형.                        |
| 아이콘 버튼       | 작은 액션·도구 모음. 라벨이 시각적으로 없을 수 있어 접근성 라벨 필수(§5). |
| 녹음 버튼         | **녹음 시작·정지 전용.** 다른 액션에 차용하지 않는다.                     |
| 인라인 링크·버튼  | 보조·메타 위계.                                                           |

**경계**

- 새 컴포넌트를 만들기 전에, 위 4종과 카드의 조합으로 표현 가능한지 먼저 확인한다.
- 새 버튼 변형을 만들지 않는다. 변형은 **강도(primary/outline/ghost) 축으로만** 분기한다.

## 2. 상태 표시

화면이 바뀌는 모든 자리에 **로딩·빈·비활성·진행·오류·경고** 여섯 상태를 의식한다.

| 상태   | 표시 방식                                                          |
| ------ | ------------------------------------------------------------------ |
| 로딩   | 짧은 한국어 한 줄 + 보조 텍스트 색. 스피너로 가득 채우지 않는다.   |
| 빈     | 한 문장 안내 + 다음 행동 힌트. 그래픽·일러스트는 쓰지 않는다.      |
| 비활성 | 흐려짐 + 막힘 표시 + **사유를 함께 안내**(왜 비활성인지).          |
| 진행률 | 원형 트랙 + 시스템 액센트 채움. 백분율 숫자는 의미 있을 때만 함께. |
| 오류   | 붉은 텍스트 + 의미 역할은 알림(§5).                                |
| 경고   | 호박색 텍스트만. 배경 면적으로 강조하지 않는다.                    |

**경계**

- 같은 영역에 상태 두 개가 동시에 보이지 않게 한다. 우선순위는 **오류 > 경고 > 로딩 > 빈**.

---

## 3. 토스트

- 토스트는 **페이지 상단 가운데** 고정 위치, 색상 톤으로 의미를 구분한다.
- **결과를 알릴 때만** 띄운다(예: "회의록이 완성되었습니다"). 액션은 한 개까지("바로 보기").
- 정보·중간 진행·반복 알림은 토스트로 띄우지 않는다 — 인라인 상태로 표시한다.
- 사용자가 토스트를 놓쳐도 **같은 정보가 화면 어딘가에 남아 있어야** 한다(목록·상세에서 다시 발견 가능).

---

## 4. 모션

- **변하는 속성만** 트랜지션을 건다. 페이지 페이드·의미 없는 호버 애니메이션은 쓰지 않는다.
- 시간 기준
  - 상태 전환·라운드 변화: 보통 길이의 부드러운 곡선.
  - 미터·진행률: 짧고 선형.
- 사용자가 **모션 감소(reduced motion)** 를 선호하면 트랜지션을 제거한다.

---

## 5. 접근성

- 아이콘만으로 의미를 가진 컨트롤은 **한국어 접근성 라벨** 필수.
- 동적으로 갱신되는 영역(스크립트·진행 안내)은 **조용히(polite) 알리는 라이브 영역**으로 둔다.
- 의미는 의미 역할로 명시한다 — 알림(오류), 상태(진행 안내), 진행률, 미터.
- **키보드 포커스 링은 시스템 액센트 한 색으로 통일**한다([DESIGN.md](./DESIGN.md) §2 참조).
- 문서 언어는 한국어로 고정.
