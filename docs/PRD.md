# Whirr — 제품 요구사항 (PRD)

## 한 줄

데스크톱 브라우저에서 회의 맥락과 함께 음성을 녹음하고, **스크립트와 AI 요약**을 만든 뒤 세션을 **로컬에만** 저장·열람한다.

## 목표

| 구분   | 내용                                                             |
| ------ | ---------------------------------------------------------------- |
| 1순위  | 녹음 전 맥락 입력 → 녹음 → 스크립트·AI 요약 생성 → 세션 자동 저장 |
| 2순위  | 날짜 기준 과거 세션 목록·상세 열람                               |
| 비목표 | 서버 텍스트 저장, 다국어 자동 감지, 모바일, 다중 사용자·계정      |

## 사용자·전제

- 단일 사용자, **로그인 없음**.
- 주 언어 한국어(영어 전문 용어 혼용 가능).
- 네트워크 전제(오프라인 STT 불필요).

## 사용 흐름(개념)

1. 사용자가 노트 제목, 참석자, 주제, 키워드, 요약 형식을 필요하면 먼저 채운다.
2. 사용자가 녹음을 시작·종료한다.
3. 녹음 후 스크립트와 AI 요약이 만들어지고, 세션이 **클라이언트 저장소**에 남는다.
4. 목록 화면에서 날짜 그룹·제목·상세 열람이 가능하다.

## 기능 요구(MVP)

- **녹음**: 브라우저 마이크 캡처. 오디오는 전사·다운로드 보조 용도 외에 앱 서버가 영구 보관하지 않음.
- **노트 맥락**: 제목, 참석자, 주제, 키워드, 요약 형식을 녹음 전에 입력할 수 있음.
- **전사·요약**: 기본 제품 흐름은 녹음 후 일괄 처리. 개발·교체 가능성 범위에서 실시간 계열도 같은 화면 흐름에 매핑된다.
- **저장**: 스크립트, 요약, 노트 맥락을 로컬 DB에 보관. MVP에서 삭제 미지원.
- **목록**: 최신순, 날짜별 그룹, 제목 우선 표시, 상세 보기.

## 비기능·보안·개인정보(요지)

- 데스크톱 브라우저 위주(Chrome 우선).
- 녹음 시작, 전사·요약 지연, 로컬 저장에 대한 **목표 임계**는 제품 품질 기준으로 유지한다.
- 오디오는 전사 목적 외부 전송에만 쓰이며, 텍스트는 **앱 서버에 상주하지 않음**.
- 전 구간 암호화 전송. API 비밀은 **클라이언트에 두지 않음**(서버 경유 발급 등으로 보호).
- 접근성: 키보드 조작·스크린 리더 고려.
