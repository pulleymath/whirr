# Whirr — 제품 요구사항 (PRD)

## 한 줄

데스크톱 브라우저에서 음성을 녹음하고, **외부 STT로 실시간 전사**한 뒤 텍스트를 **로컬에만** 저장·열람한다.

## 목표

| 구분   | 내용                                                         |
| ------ | ------------------------------------------------------------ |
| 1순위  | 녹음 → 실시간 전사 표시 → 텍스트 자동 저장                   |
| 2순위  | 날짜 기준 과거 세션 목록·상세 열람                           |
| 비목표 | 오디오 파일 보관, 다국어 자동 감지, 모바일, 다중 사용자·계정 |

## 사용자·전제

- 단일 사용자, **로그인 없음**.
- 주 언어 한국어(영어 전문 용어 혼용 가능).
- 네트워크 전제(오프라인 STT 불필요).

## 사용 흐름(개념)

1. 사용자가 녹음을 시작·종료한다.
2. 오디오는 STT로 스트리밍되고, 부분·최종 전사가 UI에 반영된다.
3. 녹음 종료 시 세션(시각·전사 텍스트)이 **클라이언트 저장소**에 남는다.
4. 목록 화면에서 날짜 그룹·미리보기·상세 열람이 가능하다.

## 기능 요구(MVP)

- **녹음**: 브라우저 마이크 캡처, 탭 활성 시에만 동작. 오디오는 **영구 저장하지 않음**.
- **전사**: 기본은 **실시간**(부분 결과) + 종료 시 확정. 설정에서 **녹음 후 일괄 전사(batch)** 를 선택하면 녹음을 마친 뒤 한 번에 REST 전사한다(실시간 미리보기 없음). 엔진·프로토콜은 [ARCHITECTURE.md](./ARCHITECTURE.md)·[DECISIONS.md](./DECISIONS.md) 범위에서 교체 가능해야 함.
- **저장**: 전사 텍스트만 로컬 DB에 보관. MVP에서 삭제 미지원.
- **목록**: 최신순, 날짜별 그룹, 미리보기, 상세 보기.

## 비기능·보안·개인정보(요지)

- 데스크톱 브라우저 위주(Chrome 우선).
- 전사 지연·시작 지연·로컬 저장에 대한 **목표 임계**는 제품 품질 기준으로 유지한다.
- 오디오는 전사 목적 외부 전송에만 쓰이며, 텍스트는 **앱 서버에 상주하지 않음**.
- 전 구간 암호화 전송. API 비밀은 **클라이언트에 두지 않음**(서버 경유 발급 등으로 보호).
- 접근성: 키보드 조작·스크린 리더 고려.
