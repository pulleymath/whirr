# Whirr — Product Requirements Document

---

## 1. 제품 개요

웹 브라우저에서 음성을 녹음하고, **실시간으로 텍스트를 전사**하여 화면에 표시하며, 전사 결과를 **자동으로 기록·보관**하는 웹 애플리케이션이다.

사용자는 별도의 설치 없이 브라우저만 열면 바로 음성 메모를 시작할 수 있다. 녹음이 끝나면 전사된 텍스트가 자동 저장되어, 날짜별로 과거 기록을 열람할 수 있다.

---

## 2. 목표

| 구분       | 내용                                                         |
| ---------- | ------------------------------------------------------------ |
| **1순위**  | 웹에서 녹음 → 실시간 전사 표시 → 텍스트 자동 저장            |
| **2순위**  | 날짜별 세션 목록으로 과거 기록 열람                          |
| **비목표** | 오디오 파일 저장, 다국어 자동 감지, 모바일 지원, 다중 사용자 |

---

## 3. 사용자

- **단일 사용자**: 웹에 접속한 본인만 사용한다.
- **로그인 없음**: 인증·계정 시스템은 두지 않는다.
- **언어**: 한국어가 주 언어이며, 전문 용어에 영어가 혼입될 수 있다.

---

## 4. 핵심 사용 흐름

```
브라우저 접속 → [녹음 버튼 클릭] → 마이크 캡처 시작
      ↓
오디오 청크를 STT API(AssemblyAI)에 WebSocket으로 직접 전송
      ↓
STT API가 전사 결과를 실시간 반환 (부분 텍스트)
      ↓
프론트엔드에 전사 텍스트가 실시간으로 표시됨
      ↓
[녹음 중지 버튼 클릭] → 최종 전사 텍스트 확정
      ↓
세션(날짜+시간, 전사 텍스트)을 IndexedDB에 자동 저장
```

### 4.1. 녹음 화면

- 중앙에 **녹음 시작/중지 버튼**
- 녹음 중: 오디오 레벨 시각 피드백(웨이브폼 또는 레벨 바), 경과 시간 표시
- 전사 텍스트가 **실시간으로 아래에 나타남** (단어/구절 단위로 갱신)
- 녹음 종료 시: 최종 텍스트 확정, 세션 자동 저장

### 4.2. 세션 목록 화면

- **날짜별로 그룹화**된 과거 세션 목록
- 각 항목: 날짜·시각, 전사 텍스트 미리보기(앞 N자)
- 항목 클릭 시 전체 전사 텍스트 열람

## 5. 기능 요구사항 (MVP)

### F1. 웹 녹음

| 항목        | 내용                                                |
| ----------- | --------------------------------------------------- |
| 오디오 캡처 | 브라우저 `getUserMedia` API로 마이크 입력           |
| 동작 조건   | **웹 탭이 열려 있는 상태**에서만 녹음 가능          |
| 오디오 포맷 | 16kHz, mono, PCM 16-bit (또는 백엔드와 합의한 포맷) |
| 오디오 저장 | **저장하지 않음** — 전사 완료 후 오디오 데이터 폐기 |

### F2. 실시간 전사

| 항목      | 내용                                                             |
| --------- | ---------------------------------------------------------------- |
| 전송 방식 | 오디오 청크를 **WebSocket**으로 STT API에 직접 스트리밍          |
| 전사 엔진 | **AssemblyAI** (Whisper Streaming) — 추후 교체 가능하도록 추상화 |
| 부분 결과 | STT API에서 push된 중간 결과를 프론트엔드에 **실시간 표시**      |
| 최종 결과 | 녹음 종료 시 잔여 오디오까지 전사하여 최종 텍스트 확정           |
| 언어      | 한국어 기본, 영어 전문 용어 혼용 허용                            |

### F3. 세션 저장

| 항목        | 내용                                       |
| ----------- | ------------------------------------------ |
| 저장 위치   | 브라우저 **IndexedDB** (클라이언트 로컬)   |
| 저장 데이터 | 세션 ID, 생성 시각, 전사 텍스트 (텍스트만) |
| 저장 시점   | 녹음 종료(전사 확정) 시 자동 저장          |
| 삭제        | **MVP에서 미지원** (Phase 2)               |

### F4. 세션 목록

| 항목      | 내용                     |
| --------- | ------------------------ |
| 정렬      | 최신 순                  |
| 그룹화    | 날짜별                   |
| 미리보기  | 전사 텍스트 앞부분 표시  |
| 상세 보기 | 클릭 시 전체 텍스트 열람 |

## 6. 비기능 요구사항

### 6.1. 플랫폼·브라우저

- **데스크톱 브라우저만** 지원 (Chrome 최우선, Safari·Firefox 가능하면 지원)
- 오프라인 모드 불필요 (네트워크 STT 전제)

### 6.2. 성능

| 지표           | 목표                                  |
| -------------- | ------------------------------------- |
| 전사 지연      | 발화 후 **3초 이내** 부분 텍스트 표시 |
| 녹음 시작 지연 | 버튼 클릭 후 **1초 이내** 캡처 시작   |
| IndexedDB 저장 | 세션당 **500ms 이내**                 |

### 6.3. 개인정보·보안

- 오디오 데이터는 전사 목적으로만 외부 STT API(AssemblyAI)에 전송되며, API 제공자의 데이터 정책에 따라 처리됨
- 전사 텍스트는 **서버에 저장하지 않음** — IndexedDB(클라이언트)에만 존재
- 모든 통신은 **HTTPS/WSS** 사용
- API 키는 클라이언트에 노출되지 않도록 **임시 토큰** 방식 사용

### 6.4. 접근성

- 키보드로 녹음 시작/중지 가능
- 스크린 리더 호환 고려 (ARIA 라벨)

---

## 7. 범위 정의

### MVP (Phase 1)

1. 웹 녹음 (마이크 캡처 + 오디오 스트리밍)
2. 실시간 전사 표시 (AssemblyAI WebSocket → 부분 텍스트 push)
3. 세션 자동 저장 (IndexedDB, 텍스트만)
4. 날짜별 세션 목록 + 상세 보기

### Phase 2 (후속)

- 세션 삭제·편집
- 텍스트 내보내기 (txt, 클립보드 복사)
- 전문 용어 사전 (custom words)
- 사운드 피드백
- 세션 검색

## 8. 결정 사항

기술 선택의 상세 근거는 `DECISIONS.md`를 참조한다.

| 항목                  | 결정                                                 | 비고                                             |
| --------------------- | ---------------------------------------------------- | ------------------------------------------------ |
| 프로젝트 이름         | **Whirr**                                            |                                                  |
| STT 엔진              | **AssemblyAI** (Whisper Streaming)                   | 추후 Deepgram·자체 호스팅 전환 가능하도록 추상화 |
| 프론트엔드 프레임워크 | **Next.js** (App Router)                             |                                                  |
| 별도 백엔드 서버      | **없음** — Next.js API Route로 토큰 발급만 처리      | 브라우저가 STT API에 직접 WebSocket 연결         |
| 배포 환경             | **Vercel** (프론트엔드 + API Route)                  |                                                  |
| 부분 결과 갱신 단위   | AssemblyAI의 `PartialTranscript` / `FinalTranscript` | API가 제공하는 단위 그대로 사용                  |

### 미결정 사항

| 항목                       | 선택지        | 비고                           |
| -------------------------- | ------------- | ------------------------------ |
| 오디오 청크 크기·전송 간격 | 100ms ~ 250ms | 전사 지연 vs 부하 트레이드오프 |
